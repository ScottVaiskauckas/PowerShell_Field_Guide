Foreach loops

In the last lesson, I ended with an example of how to iterate through an array using a for loop and said that PowerShell provides a special kind of loop called a Foreach loop for just this purpose.


Example1

In our first example, we start by initializing an array with 4 elements named $ATXBirdArray and assign the strings "bluejays", "cardinals", "mocking birds", and "monk parakeets" on line 1. On line 3 we have our Foreach loop. We are going to iterate over the items in the $ATXBirdArray and use the $Bird variable as a reference to that item. PowerShell enters the loop and writes "You can find bluejays in Austin, Texas!" to the console. PowerShell reaches the end of the loop and moves on to the next item in the array. PowerShell writes "You can find cardinals in Austin, Texas!" to the console. PowerShell once again reaches the end of the loop and moves on to the next item in the array. PowerShell now writes "You can find mocking birds in Austin, Texas!" to the console. PowerShell reaches the end of the loop and moves on to the next item in the array. PowerShell writes "You can find monk parakeets in Austin, Texas!" and exits the loop.

Obligatory Foreach-Object alias Foreach warning

Every course on PowerShell I have ever seen has a warning that Foreach-Object has an alias named Foreach. It's almost impossible to accidentally use one in place of the other in practice. I'm mentioning it here for completeness, and to give you a handy rule of thumb. If you're piping commands into foreach, it is Foreach-Object. If you're not piping commands into foreach, you're using the foreach loop construct.
