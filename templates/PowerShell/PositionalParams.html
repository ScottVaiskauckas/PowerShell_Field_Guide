Parameters

Parameters are another, more commonly used way to provide input to scripts and cmdlets in PowerShell than using the Read-Host cmdlet. Parameters allow you to provide values to a script when you call the script in the shell that will make the script do different things. The values that you provide the parameters are called arguments.

Positional Parameters


There are two types of parameters in PowerShell, positional parameters and named parameters. I will be covering positional parameters here and named parameters in the next lesson.

The syntax for a postional parameter is simple. All you have to do is reference the $args array in your code and you are good to go. That's because parameters use the position of an argument to determine which parameter a programmer intended to use. This will be more obvious when you can contrast positional parameters against named parameters (covered next lesson).
