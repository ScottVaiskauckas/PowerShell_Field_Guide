Storing data in csv files

This is the second lesson in a series of lessons on how you can save data so you can use it later with PowerShell.

Before you begin:

In this lesson, we will be using PowerShell files you can find in the following github repo in a folder named PowerShell_Peristance: https://github.com/ScottVaiskauckas/PowerShell. If you have never used Github before, for full instructions on how to download this file, please see my page on Getting Started.

Csv files

Csv files are files that use commas and returns to represent data in tabular format. The commas seperate table columns and the returns represent rows. The first row contains the table header. You may have worked with these files before in Excel or other spreadsheet software. PowerShell makes working with Csv files very easy. This is great for generating reports since you can easily convert the file to .xlsx format for the technically challenged.

Export-Csv


The Export-Csv cmdlet will let you conver PowerShell objects to csv strings and save those strings to a csv file. The strings retain the properties of the object, but lose the object's methods. Here you can see just how easy it is to save the output from Get-Timezone by piping it into the Export-Csv cmdlet.

Import-Csv


The Import-Csv cmdlet will let you import the data you exported with Export-Csv back into the shell. We can run Import-Csv and see our exported data has been preserved.

Out-Gridview

If you have a spreadsheet application like Excel then you can open csv files and view them the same as xlsx files. If you don't have one installed on a machine, you can use the Out-Gridview cmdlet to display the data in a table format.
